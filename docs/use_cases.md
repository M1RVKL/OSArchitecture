# ⚙️ Use Cases

## 1. Створення замовлення
* **Актор:** Клієнт
* **Передумови:** Клієнт авторизований, у кошику є доступні страви з одного закладу.
* **Основний сценарій:**
  1. Клієнт ініціює створення замовлення.
  2. Система перевіряє доступність страв.
  3. Система розраховує загальну вартість.
  4. Створюється запис у таблиці `orders` (статус `CREATED`).
  5. Поточні ціни фіксуються в таблиці `order_items`.
* **Альтернативні сценарії:** Якщо страва стала недоступною, система повертає помилку та пропонує оновити кошик.

## 2. Прийняття замовлення кур'єром
* **Актор:** Кур'єр
* **Передумови:** Замовлення має статус `READY` або `PREPARING` і не має призначеного кур'єра.
* **Основний сценарій:**
  1. Кур'єр обирає замовлення зі списку доступних.
  2. Система перевіряє, чи замовлення ще вільне.
  3. Система призначає `courier_id` та оновлює статус на `DELIVERING`.
* **Альтернативні сценарії:** Якщо замовлення вже прийняв інший кур'єр (конкурентність), система повідомляє про це та оновлює список.

## 3. Керування меню
* **Актор:** Менеджер ресторану
* **Передумови:** Менеджер авторизований і прив'язаний до конкретного ресторану.
* **Основний сценарій:**
  1. Менеджер додає нову страву (назва, ціна).
  2. Система валідує дані та створює запис у `menu_items`.
  3. Страва стає доступною для клієнтів.
